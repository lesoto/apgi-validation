# APGI Data Repository

This directory contains data files and metadata for the APGI validation framework.

## Directory Structure

```
data_repository/
├── raw_data/           # Raw neurophysiological data
│   ├── eeg/            # EEG recordings
│   ├── fmri/           # fMRI scans
│   └── behavioral/     # Behavioral task data
├── processed_data/     # Processed and analyzed data
│   ├── behavioral/     # Processed behavioral metrics
│   └── neural/         # Processed neural signatures
├── metadata/           # Dataset metadata and documentation
└── codebooks/          # Data processing codebooks
```

## Data Access

### Using Real Data

When real data becomes available, follow these steps:

1. Download the dataset from OpenNeuro
2. Extract to the appropriate subdirectory in `raw_data/`
3. Update the protocol configuration to point to the real data files
4. Run the protocol with real data

### Synthetic Data

Synthetic data is generated by the validation protocols when real data is not available. This is the default mode for most protocols.

## Data Processing Pipeline

1. **Raw Data Loading**: Load raw data from OpenNeuro format
2. **Preprocessing**: Apply standard preprocessing (filtering, artifact removal, etc.)
3. **Feature Extraction**: Extract relevant features for validation
4. **Quality Control**: Verify data quality and exclude bad trials
5. **Storage**: Save processed data for reproducibility

## Metadata Standards

Each dataset should include:

- **Provenance**: Source, collection method, consent information
- **Preprocessing**: All preprocessing steps applied
- **Quality Metrics**: Signal-to-noise ratio, artifact rejection rates
- **Documentation**: Experimental design, task parameters

## Data Citation

When using data from this repository, please cite both:

1. The original dataset source (OpenNeuro)
2. The APGI validation framework
